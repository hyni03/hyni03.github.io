gitblog에 Google analytics 설정하기

google analytics를 Github 홈페이지에 적용하는 방법에 대한 설명글

Nov 28, 2022 About 2 mins

#Github#Google#Analytics

Google Analytics란?

: 구글에서 무료로 제공하고 있는 웹분석 서비스




Google analytics 설정하기


1. Google analytics 계정 생성

google anayltics 바로가기


1.사이트에 접속하여 측정 시작 누르기



2-1. 계정 설정하기


개인 이름 설정하기 (직접적인 영향은 없다)



2-2. 속성 설정하기

속성 이름에 블로그 주소와 시간대를 대한민국, 통화는 대한민국 원으로 설정


2-3. 비지니스 정보는 필요에 따라 설정(건너뛰어도 된다) 후 만들기




2. 블로그와 연결 및 측정ID 확인


1.analytics 계정과 블로그 연결

데이터 스트림에 들어가 웹 선택


블로그 주소와 이름 작성하기



2.측정 ID 확인하기

측정ID는 코드 작성 시 필요하므로 기억해두기




3. 코드 수정하기


1.태그 설치하기

analytics 기능을 사용하기 위해 태그 설치가 필요 데이터 스트림에서 태그 안내 보기 선택 후 직접 설치 클릭. 코드 복사하기



복사한 코드를 /_includes 폴더안에 analytics.html 파일을 생성하여 붙여넣기



2.default 파일 수정
_layouts 폴더에 있는 default.html의 body 맨 마지막에 다음 코드 추가

<!-- Google tag (gtag.js) -->
<script async src="https://www.googletagmanager.com/gtag/js?id=G-9JDJSLD96W"></script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());

  gtag('config', 'G-9JDJSLD96W');
</script>


3.config.yml 수정하기
config.yml에 다음 코드 추가하기

# Analytics
analytics:
  provider: "google-gtag"
  google:
    tracking_id: "측정ID"




4. 측정 확인하기


위에 과정들을 다 완료하면 블로그와 analytics가 정상적으로 연결됐을 것입니다.
다시 analytics에 접속해보면 방문자 수가 카운팅됩니다.
